You're trying to open the HTML file directly in the browser, which causes CORS (Cross-Origin Resource Sharing) errors. The web app needs to run through a web server to work properly.

How to Fix This:

  1. Option 1: Use the Desktop App (Recommended)
    python3 qr_generator_chromebook.py
  2. Option 2: Run the Web Server
    python3 web_qr_generator.py
    Then open: http://localhost:5000
  3. Option 3: Use the Launcher Script
    python3 run_web_app.py
  4. Option 4: Windows Users
    run_web_app.bat
Note: The web app requires a web server to handle API requests. Opening HTML files directly in the browser won't work due to browser security restrictions.

Setup Instructions for Chrome OS:

  1. Enable Linux on your Chromebook (Settings → Advanced → Developers)
  2. Open Terminal (Ctrl + Alt + T, then type 'shell')
  3. Install Python: sudo apt install python3 python3-pip
  4. Install dependencies: pip3 install flask qrcode[pil] requests
  5. Run the app: python3 qr_generator_chromebook.py

Common Issues:

  • Python not found: Use python3 instead of python
  • Module not found: Install dependencies with pip3 install -r requirements.txt
  • Permission denied: Make script executable: chmod +x qr_generator_chromebook.py
  • Port already in use: Change port in web_qr_generator.py or kill existing process
